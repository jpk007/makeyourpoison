// architecture

How it works

Every ISO is built entirely from scratch inside an isolated Docker container.

1

Config saved

Frontend POSTs your config → PostgreSQL → returns a configId.

2

Job queued

Build job added to Bull/Redis queue with a UUID job ID. Max 2 concurrent builds.

3

Docker container spins up

Uses pre-compiled vmlinuz (Linux 6.6.22 from source) and busybox.

4

Rootfs assembled

Alpine apk bootstrapped, desktop + packages installed, initramfs built with live-boot support.

5

ISO ready

xorriso assembles a dual BIOS+UEFI bootable ISO. Logs stream live via WebSocket.

🏁 Milestone 1 — 29 March 2026

First live boot

The first ISO built entirely from scratch booted successfully inside Hyper-V on Windows 11. Full boot chain confirmed working end-to-end.

  • Linux 6.6.22 kernel compiled from kernel.org source
  • Hyper-V drivers loaded from initramfs (hv_vmbus → hv_storvsc)
  • squashfs found and mounted from /dev/sr0
  • overlayfs — read-only squashfs + writable tmpfs layer
  • switch_root into live Alpine userspace
  • Full filesystem tree confirmed: bin dev home lib proc root sys usr var
